Machine learning workflows as code

dstack makes it very easy to define ML workflows and run them on any cloud platform. It provisions infrastructure, manages data, and monitors usage for you.

Ideal for processing data, training models, running apps, and any other ML development tasks.

Quick start Learn more

Features

Workflows as code

ML workflows can be defined as YAML or Python code and executed locally or on any cloud provider with a single command, without custom Docker images or Kubernetes.

Infrastructure provisioning

Automatically provision the required cloud resources at workflow startup and tear them down after workflow completion, irrespective of the cloud provider used.

Dev environments

Create dev environments such as IDEs or notebooks with the required cloud resources and dependencies with a single command. Or attach your own IDE to running workflows.

Applications

Run machine learning applications with the required cloud resources and dependencies using a single command, and access them as if they were running locally.

Data management

Save the output artifacts from machine learning workflows and reuse them as dependencies in other workflows. Artifacts can be used to reuse data, models, and environments.

Workflow scheduling

The built-in scheduling capabilities allow for handling the queue of running workflows, efficient use of cloud spot instances, and running workflows on a schedule.

Usage monitoring

Monitor compute usage costs. Define and enforce compute usage quotas per workflow or per project. Manage compute usage policies per project.

Roles and permissions

The built-in authentication allows you to manage user profiles, their roles, and permissions across multiple projects.

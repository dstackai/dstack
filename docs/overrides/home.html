Cost-effective LLM development

dstack is an open-source tool that simplifies LLM development across multiple clouds.

It streamlines development and deployment, reduces cloud costs, and frees users from vendor lock-in.

Get started

Leverage multiple clouds to cut costs

Dev environments

Configure the hardware resources, such as GPU, memory, specify your preference for using spot instances.

dstack automatically provisions cloud resources, fetches your code, and forwards ports for secure access.

Access the cloud dev environment conveniently using your local desktop IDE.

Serving

Serve models using Gradio, Streamlit, FastAPI, vLLM, and other frameworks in any cloud.

Configure the hardware resources you need (GPU, memory, etc.) and indicate whether you want to use spot or on-demand instances.

dstack will automatically provision cloud resources, and forward ports for secure and convenient access.

Training

Pre-train and finetune your own state-of-the-art models easily and cost-effectively in any cloud.

Have cloud resources automatically provisioned based on your configuration.

Access your data and store output artifacts using declarative configuration or the Python SDK.

Run LLM workloads easily and cost-effectively

Faster iterations

Define any workloads declaratively as code. Run them on demand using the CLI, eliminating the need for cloud UI or proprietary APIs.

Cost optimization

Save costs by effectively utilizing cloud spot instances across multiple regions. Automatically recover from interruptions. Track GPU usage, while enforcing limits per team.

Data-centric approach

Run dev environments and tasks where your data is located. Eliminate costs associated with ingress and egress data transfers. Ensure the security and privacy of your data and models.

No MLOps is required

dstack integrates with your cloud natively and doesn't require engineering. No hustle with Terraform, Kubernetes or custom Docker images.

Cloud-agnostic

Use the same interface to work with any major cloud provider, including AWS, GCP, Azure, and Lambda. Use multiple clouds to optimize costs.

Self-hosted

Run the dstack server on your local machine or easily deploy it, as a Docker image, anywhere within your environment, including your own cloud.

Get started in less than a minute

$ pip install "dstack[aws,gcp,azure,lambda]"
$ dstack start

And you're done! Now you can run dev environments and tasks both locally and in your cloud.
